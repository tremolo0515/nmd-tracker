# ポケスリ ニュームーンデー出現記録アプリ 設計書作成依頼

## プロジェクト概要

ポケモンスリープの月次イベント「ニュームーンデー」は毎月3日間開催される。
期間中、幻のポケモン（現在はミュウ・ダークライ）が1日1回目の睡眠リサーチでのみ出現する。
ユーザーがセルをタップするだけで出現可否を記録し、次回の出現確率を表示するWebアプリ。

## 技術スタック

- Next.js（App Router）+ TypeScript + Tailwind CSS
- データ永続化：localStorage
- ホスティング：Vercel

## 確率モデル（最重要）

出現確率はポケモンごとに独立した「連続ハズレ日数N」で決まる。

P = (N+1)(N+2) / 12

| N（直近の連続ハズレ日数） | 出現確率 |
|---|---|
| 0 | 16.7%（1/6）|
| 1 | 50.0%（3/6）|
| 2 | 100%（6/6）|

### Nの更新ルール

- 出現○ → N = 0 にリセット
- ハズレ× → N = N + 1
- N=2で必ず出現するためN=3は存在しない
- **月をまたいで引き継ぐ**。月次リセットはない

### 月跨ぎの例

4月のイベント結果が「×・×・○（N=2で確定出現）」の場合、5月1日目はN=0（16.7%）。
4月のイベント結果が「○・×・-（3日目未到達）」の場合、5月1日目はN=1（50%）。
4月のイベント結果が「×・○・×」の場合、5月1日目はN=1（50%）。

## 機能要件

### 記録
- 月ごとに3つのセルを表示（1日目・2日目・3日目）
- セルの状態：未記録 / 出現○ / ハズレ×（タップで順に切り替え）
- ポケモンごとにカードを1枚表示（現在2枚：ミュウ・ダークライ）
- 開催日の管理は不要。月ごとに3セルあればよい

### 確率表示
- 各セルの右または下に「このセルでの出現確率 XX%」を表示
- 記録済みセルは実績を表示、未記録セルは現在のNから計算した確率を表示

### 拡張性（最重要）
- ポケモンの追加は設定ファイル1箇所の変更のみで完結すること
- UIもデータ構造もポケモン数に依存しない設計にすること

```ts
// pokemon.config.ts のイメージ
export const POKEMON_CONFIG = [
  { id: "darkrai", name: "ダークライ", color: "#6B3FA0" },
  { id: "mew",     name: "ミュウ",     color: "#C2185B" },
] as const;
```

## 依頼する成果物

以下を含む設計書を作成してください。

1. **データ構造**
   - localStorageのキー設計とスキーマ（型定義含む）
   - ポケモンごとの全履歴（月跨ぎ対応）の保持方法

2. **確率計算ロジック**
   - N算出の関数設計（全履歴からNを導出する処理）
   - セル状態変更時の再計算フロー

3. **コンポーネント構成**
   - ページ・カード・セル等の分割方針
   - propsの型定義

4. **ファイル構成**
   - Next.js App Router前提のディレクトリ構成

5. **拡張手順**
   - 新ポケモン追加時の変更箇所を明記（設定ファイル以外に触る場所がないことを保証）
