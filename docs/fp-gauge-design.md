# フレンドポイントゲージ機能 設計書

Last-modified: 2026-05-19

## 概要

ポケモンスリープのおやつタイムで管理する「フレンドポイント（FP）」の進捗を、既存の幻ポケモン出現トラッカーに統合して記録・可視化する機能。

---

## ゲーム内仕様（実装の前提知識）

### フレンドポイントの基本ルール

- 各ポケモン種族ごとに必要FPが設定されており、MAXまで貯めると仲間になる
- FPは**種族単位**で管理される（同種が複数出現しても共有）
- FPはリセットされず、次回以降のおやつタイムに引き継がれる
- 仲間になると FP は 0 にリセット（次のフレンドレベルへ）
- 超過分は切り捨て（1枚のサブレで2レベル以上上がることはない）

### サブレ別FP上昇量

| サブレ種別 | 通常FP上昇 | 大成功（3倍） | 超成功（MAX） |
|------------|-----------|--------------|--------------|
| ポケサブレ  | +1        | +3           | MAX          |
| スーパーサブレ | +3     | +9           | MAX          |
| ボーナスサブレ | +3 or +4     | +9 or +12           | MAX          |
| マスターサブレ | MAX    | —            | —            |

### 必要FP（幻のポケモン）

本ツールの対象ポケモン（幻のポケモン）の必要FPは **25**。

### 出現時の自動FP加算

幻のポケモンは月3回の新月期間に低確率で出現し、出現した際はゲーム内仕様として **ボーナス +3 FP** が付与される。そのため、ツールでも「appeared」記録時に **+3 FP** を自動加算する。

> 注: これはサブレによるFP加算とは別のボーナスであり、出現した事実に対して固定で付与される。

## データ設計

### ストレージキー

```
"nmd-tracker-fp"  ← 既存の "nmd-tracker-data" とは別キー
```

### 型定義

```typescript
type FpData = Record<string, FpEntry>  // キー = pokemon.id

interface FpEntry {
  current: number   // 現在のFP（0〜max）
  max: number       // 必要FP上限（PokemonConfig.maxFp から取得）
}
```

`caughtCount`（仲間にした累計）は将来拡張として保留。現バージョンでは「仲間にした」ボタンで current を 0 にリセットするのみ。

### PokemonConfig への追加フィールド

```typescript
// config/pokemon.ts
export interface PokemonConfig {
  // ...既存フィールド...
  maxFp: number  // 必要フレンドポイント上限
}

// 各ポケモンの設定値
// ダークライ: maxFp: 25（幻のポケモン）
// ミュウ:     maxFp: 25（幻のポケモン）
```

### データの不変条件

- `current` は常に `0 ≤ current ≤ max`
- `max` は `PokemonConfig.maxFp` と常に一致させる（設定変更時は localStorage を更新）

## UI設計

### 全体レイアウト

**ゲージ記録モード** は OFF がデフォルト。ヘッダーのアイコンボタンで ON/OFF を切り替える。

```
ヘッダー:  🌙 Mythical Pokémon Tracker    [❤️ゲージ] [📷シェア]
```

モード ON 時、左列ポートレートの下端にゲージ + ボタンを追加表示する。

### ポートレート行の構造（モード ON 時）

```
┌────────────────────────┐
│  累計出現率（既存）      │
│  ポートレート画像（既存） │   ← 高さ変更なし（ROW_H を維持）
├────────────────────────┤
│ [−]  ●●●●○○○○○○  [+]  │   ← FP ゲージ行（高さ追加）
└────────────────────────┘
```

ポートレートとゲージ行を縦に並べる。ROW_H に FP 行分（約 28px）を上乗せする。

### FPゲージの仕様

| 項目 | 仕様 |
|------|------|
| 表示幅 | 左列幅 88px から左右ボタン（各 16px）と余白を引いた幅 |
| セグメント数 | `maxFp` と同じ個数（幻ポケモンは 25 個） |
| セグメント形状 | 楕円形の粒（pill 型）|
| 塗りつぶし色 | ポケモンの `accentColor` |
| 未入力色 | `rgba(30, 41, 59, 0.8)`（slate-800 系） |
| セグメント間隔 | 1px gap |
| ボタン | `−` / `+`、タップ領域は最低 24px × 24px |

### 増減ボタン動作

| 操作 | 動作 |
|------|------|
| `+` タップ | `current = Math.min(current + 1, max)` |
| `−` タップ | `current = Math.max(current - 1, 0)` |
| `current === max` | `+` ボタンを無効化（視覚的にも dim 表示） |
| `current === 0` | `−` ボタンを無効化（視覚的にも dim 表示） |
| `current === max` 時に長押し | 「仲間にした！」確認 → `current = 0` にリセット（将来拡張） |

### 出現記録との連動

```
handleStateChange(pokemonId, monthKey, day, "appeared") が呼ばれたとき:
  fpData[pokemonId].current = Math.min(current + 3, max)
  localStorage に FP データを保存
```

この自動加算はゲージ記録モードの ON/OFF に関わらず常に行う（ユーザーが後で確認できるように）。

## 状態管理

```typescript
// page.tsx に追加
const [fpData, setFpData] = useState<FpData>({})
const [fpMode, setFpMode] = useState(false)  // ゲージ記録モード
```

FP データの読み書きは既存の `data` と同じパターンで localStorage を使う。

---

## 非機能要件

- **後方互換性**: `nmd-tracker-fp` キーが存在しない場合は `{ current: 0, max: pokemon.maxFp }` をデフォルト値として使う。データ移行不要。
- **モードの永続化**: `fpMode` は localStorage に保存しない（セッションごとリセット）。起動時は常に OFF。
- **パフォーマンス**: ゲージ行は絶対配置やアニメーションを持たない軽量実装。ROW_H の変更に伴い既存スクロール位置計算が壊れないことを確認する。
