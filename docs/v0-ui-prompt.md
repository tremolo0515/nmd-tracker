# ポケスリ ニュームーンデー出現記録アプリ UIデザイン作成依頼

## あなたへの指示

v0（vercel.com/v0）でこのアプリのUIを実装してください。
まずUIのみを作成し、ロジックは後で追加します。
モックデータを使ってUIの見た目・操作感を完成させることを優先してください。

## 背景・目的

ポケモンスリープの月次イベント「ニュームーンデー」は毎月3日間開催される。
期間中、ミュウとダークライが1日1回目の睡眠リサーチでのみ出現する。
ユーザーがセルをタップするだけで出現可否を記録し、今後の出現確率を確認できるアプリ。

## 確率モデル（モックデータの参考に）

出会えていない連続日数をN（0〜2）としたとき：P = (N+1)(N+2) / 12

| N | 出現確率 |
|---|---|
| 0 | 16.7% |
| 1 | 50.0% |
| 2 | 100% |

## UI要件

### メイン画面の構成

- 月ごとに1画面
- ポケモンごとにカードを1枚ずつ表示（現在はミュウ・ダークライの2枚）
- 各カードに3日分のセルを横並びで表示（1日目・2日目・3日目）
- セルの状態：未記録（グレー）・出現○（光る）・ハズレ×（暗い）
- セルの下に「次の出現確率 XX%」を表示

### ポケモンカードの設計（拡張性重要）

ポケモンは今後追加される可能性があるため、カードは設定ファイルで管理する構造にする。
設定に新しいポケモンを1行追加するだけで画面にカードが増える設計にすること。

```js
// このような設定ファイルで管理するイメージ
const POKEMON_CONFIG = [
  { id: "darkrai", name: "ダークライ", color: "#6B3FA0", accentColor: "#C084FC" },
  { id: "mew",     name: "ミュウ",     color: "#C2185B", accentColor: "#F48FB1" },
  // 将来：{ id: "xxx", name: "XXX", color: "...", accentColor: "..." }
];
```

### デザイン方針

- **夜空・月モチーフ**のダーク系UI（ポケスリの世界観）
- ミュウ：ピンク系、ダークライ：紫・黒系
- セルタップ時に星がはじけるアニメーション
- 確率が高いほどセルやカードが輝く演出（16.7%→通常、50%→輝き強め、100%→最大輝き）
- スマホ縦画面に最適化（ボタン大きめ）

## 技術スタック

- React + Tailwind CSS
- データはモックで可（useState で状態管理）
- localStorageは後で追加するので今は不要

## 成果物

1. 上記UIを実装したReactコンポーネント
2. POKEMON_CONFIGの設定ファイル（ポケモン追加方法のコメント付き）
3. モックデータで全状態（未記録・○・×）が確認できること
