# 新しいポケモンの追加手順

ニュームーンデーに新たな幻のポケモンが追加された場合の手順。
**変更するファイルは 2 つだけ**（設定ファイルへの追記 + 画像ファイルの配置）。

---

## 手順

### 1. 画像ファイルを配置する

```
public/pokemon/<id>.png
```

- `<id>` は次のステップで決める英小文字の識別子
- 推奨: PNG 透過背景、正方形に近いサイズ
- 画像がない場合は `backgroundImage` を省略すれば黒背景になる

### 2. `config/pokemon.ts` に1エントリ追記する

`POKEMON_CONFIG` 配列の末尾に追加する。

```ts
{
  id: "jirachi",           // 必須: 英小文字・ハイフンのみ。一度決めたら変更不可。
  name: "ジラーチ",          // 必須: 画面表示名
  color: "#B8860B",        // 必須: ベースカラー（ポートレート背景グラデーション）
  accentColor: "#FFD700",  // 必須: アクセントカラー（セル発光・確率テキスト）
  backgroundImage: "/pokemon/jirachi.png", // 任意: 手順1で配置した画像パス
},
```

**これだけで完了。** 他のファイルへの変更は不要。

---

## 変更が不要な理由

| 変更箇所 | 理由 |
|---|---|
| `app/page.tsx` | `POKEMON_CONFIG` を `map` しているため、配列に追加すれば UI（ポートレート・セル行）が自動で増える |
| `components/day-cell.tsx` | ポケモン固有の情報を持たない汎用コンポーネント |
| `getNBeforeCell()` | `pokemonId` を文字列で受け取るだけでポケモンの種類に依存しない |
| `calculateProbability()` | 全ポケモン共通の確率モデルを使用 |
| localStorage データ | 新 `id` キーが存在しない月は自動で「未記録」として扱われる。既存ユーザーのデータは壊れない |

---

## カラーの選び方

ポケモンの公式カラーを参考に選ぶ。

| 項目 | 目安 |
|---|---|
| `color` | やや暗め・彩度低め（背景に馴染む） |
| `accentColor` | 明るめ・鮮やか（暗い背景で映える） |

確認方法: dev サーバーを起動して `config/pokemon.ts` を保存するとホットリロードで即確認できる。

```bash
npm run dev
```

- [ ] `id` が既存と重複していないことを確認した
- [ ] dev サーバーでポートレートとセル行が正しく表示されることを確認した

---

## 注意事項

- **`id` は変更不可。** localStorage のキーに使われるため、変更すると既存ユーザーの記録データが消える。
- 配列の**順番が表示順**になる。追加するポケモンは末尾に追記することを推奨。
- 将来ポケモンごとに確率モデルが異なる場合は、`PokemonConfig` に確率計算用のパラメータを追加し、`calculateProbability()` をそれに対応させる必要がある。
